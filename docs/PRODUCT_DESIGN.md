## 一、设计理念

> **核心原则：** 能用图就不用表格，能用颜色就不用数字，能一眼看懂就不要点击展开。

本系统面向**有投资经验但编程零基础的量化初学者**，所有复杂的量化概念都通过**可视化组件**呈现，让用户像看天气预报一样理解自己的交易系统状态。

### 设计原则

1. **图形优先** — 每个数据点都有对应的图形表达
2. **风险可见** — 任何页面都能感知到当前风险状态
3. **流程透明** — 从信号生成到执行的每一步都可视化
4. **目标驱动** — 所有决策回溯到用户设定的目标

## 十一、业务流程总览

### 完整使用路径（从零到第一笔交易信号）

```
设定目标 → 策略猎手 → 回测实验室 → 标的扫描 → 观察列表 → 风控护盾
 /goals      /hunt      /backtest     /scan      /watchlist   /risk
```

1. **设定目标** `/goals`：输入年化目标、最大回撤、资产类型、持仓周期，系统算出 Sharpe 要求和单笔风险上限。保存后自动跳到策略猎手。
2. **策略猎手** `/hunt`：根据目标从策略库 + GitHub + Claude AI 三个渠道检索策略，按匹配度排序。点击"去回测验证"携带策略名和默认标的跳转。
3. **回测实验室** `/backtest`：选择策略和标的，对历史数据跑回测，查看权益曲线、胜率、夏普比、逐笔交易记录。
4. **标的扫描** `/scan`：选一个已验证策略，扫描 S&P500 / 纳指100 找当前符合买入条件的标的，可直接加入观察列表或模拟下单。
5. **观察列表** `/watchlist`：持续跟踪关注标的的实时价格，点击任意标的可直接跳到该标的的回测页面。
6. **风控护盾** `/risk`：设定并监控风险红线（最大回撤阈值、单笔亏损上限、VIX 预警等），查看 AI 风险建议。
7. **宏观经济** `/macro`（可选）：查看 GDP / CPI / 利率 / 失业率等宏观指标，判断当前市场环境是否适合激进策略。

### 页面间数据流转

| 来源页面 | 目标页面 | 携带的数据 | 传递方式 |
|---------|---------|-----------|---------|
| `/goals` | `/hunt` | 目标对象（年化/回撤/品种/周期） | redirect |
| `/hunt` | `/backtest` | 策略 ID + 策略默认标的 | URL 参数 `?strategy=&symbol=` |
| `/scan` | `/watchlist` | 标的代码 | POST `/watchlist/add` |
| `/watchlist` | `/backtest` | 标的代码 | URL 参数 `?symbol=` |
| `/backtest` | `/scan` | — | 用户手动导航 |

### 设计决策说明

**为什么回测时还要选标的，策略不自带标的？**

策略是通用逻辑（比如"布林带均值回归"），它可以对任意标的运行。
回测页面需要用户指定标的，是因为同一策略在 SPY 和 TQQQ 上的表现可能完全不同——用户应该主动验证自己想交易的标的。

为了减少摩擦，每个策略现在配置了 `default_symbols` 字段，点"去回测验证"会自动带入策略最常用的标的（如 TQQQ/SQQQ 策略自动填 TQQQ，不再默认 SPY）。

## 十二、已知问题与修复记录

### BUG-001：点"去回测验证"策略下拉框未预选（已修复 2026-04-03）

**现象：** 从策略猎手点击"去回测验证"跳转到回测页，策略下拉框未自动选中对应策略，默认显示第一个。

**根因：** 策略库（`strategy_library.py`）的策略 ID（如 `tqqq_dip_buy`）与数据库 `StrategyConfig` 表里的 `strategy_name`（如 `tqqq_dip`）命名不一致，模板用字符串精确匹配所以匹配失败。

**修复：** 在 `backtest_page()` 路由里加了两级匹配逻辑：
1. 先精确匹配 `strategy_name` / `display_name`
2. 匹配失败则按分词 token 重叠度做模糊匹配，找得分最高的策略
后端把匹配到的 `config.id`（整数）传给模板，模板直接用 ID 比对，消除了字符串匹配的歧义。

**涉及文件：** `app/main.py` `backtest_page()` · `app/templates/qp_backtest.html`

---

### BUG-002：策略卡片"去回测验证"标的永远默认 SPY（已修复 2026-04-03）

**现象：** 无论点的是哪个策略（包括明确针对 TQQQ 的策略），跳转到回测页后标的输入框始终显示 SPY。

**根因：** `hunt_results.html` 里的链接硬编码 `symbol=SPY`，策略定义中没有默认标的字段。

**修复：**
1. 在 `strategy_library.py` 的每个策略定义里新增 `default_symbols` 字段（列表，首项为最优先标的）
2. `hunt_results.html` 链接改为读取 `s.default_symbols[0]`，无此字段时才 fallback 到 SPY

示例：`tqqq_dip_buy` 策略现在携带 `default_symbols: ["TQQQ", "QQQ"]`，点"去回测验证"后标的自动填 TQQQ。

**涉及文件：** `app/strategy_library.py`（所有策略）· `app/templates/partials/hunt_results.html`
