# 金渐成投资原则与策略体系

> 来源：金渐成公众号文章 224 篇（2024-2025），对冲基金/VC 背景，主投美股。
> 整理日期：2026-04-02
> 用途：作为量化平台的策略参考模板，可拆解为可编程规则

## 一、核心投资哲学

### P1: 在有鱼的地方钓鱼
- 选择有效市场（美股优于A股）：监管更强、工具更丰富、虹吸效应更强
- 只投有价值的产品，远离垃圾股
- 做确定性强的事，不赌小概率

### P2: 先防守，后进攻
- "不败将军比常胜将军难多了，防守是一门大学问"
- 任何时候先确保不会被淘汰出局，再考虑怎么赢
- 防守是为了更好地进攻 — 穿好盔甲才能上阵

### P3: 永不满仓
- 手上永远保留筹码（现金），确保任何时候都有能力下注
- 满仓+杠杆 = 赌博，不是投资
- 不满仓就是给自己机会

### P4: 耐心等时间给结果
- 过程无聊但必要，长期视角符合复利积累的特性
- 短线技术指标（KD/MACD/布林线）是"术"，看大势和看人才是"道"

### P5: 危机是清醒者的礼物
- "不要浪费每一次危机" — 危机 = 打折买优质资产的机会
- 熊市是甜蜜的加仓点，是财富的放大器
- 区分"基本面改变"vs"短期扰动"：河流生态被破坏 ≠ 鸭子临时搅浑了水

### P6: 学思路不抄作业
- 透过别人的投资操作看思路和逻辑，而非照搬买卖节点
- 结合自身情况找到最适合自己的方式和风格

### P7: 赚超预期的钱要清醒
- "赚的钱有太多不在我能力范围内的，远超预期和能力"
- 超额收益时更要加强风险控制和防守配置

## 二、选股三大维度

### D1: 看大势（宏观趋势）
- 站在时代趋势的正确一边（如 AI 浪潮、云计算革命）
- 判断"基本面是否改变" — 如果只是短期波动，不改变持仓逻辑
- 关注美联储利率周期、通胀趋势等宏观变量

### D2: 看人（管理层质量）
- "企业就是管理者人格的映射"
- 好管理层能让平庸公司变伟大，糟糕管理层会毁掉好公司
- 核心评估维度：**诚实可靠、踏实进取、眼光长远**
- 案例：微软在纳德拉上任后股价翻 16 倍 vs 特斯拉因马斯克争议持续减仓

### D3: 看长期（商业模式 + 基本面）
- 公司基本面、商业模式、竞争壁垒
- 中长线持仓为主，不做短线波段套利
- 只买看得懂的公司

## 三、账户分层体系（攻守分离架构）

```
┌──────────────────────────────────────────────────┐
│                 总资产配置                          │
├──────────┬──────────────┬────────────────────────┤
│  进取型   │    稳健型     │       防守型            │
│ (Attack) │  (Balanced)  │     (Defense)          │
│          │              │                        │
│ 科技龙头  │  蓝筹+成长    │ 高股息+债券ETF          │
│ 个股集中  │   混合配置    │  被动收入+保值           │
│          │              │                        │
│ 目标:    │  目标:        │  目标:                  │
│ 超额收益  │  稳定增值     │  压舱石, 40%+配置       │
└──────────┴──────────────┴────────────────────────┘
         ↑ 赚钱后高位套现部分利润 → 转入防守型 ↓
```

### 进取型账户（个人最大账户）
- **持仓排序**：NVDA(32%+) > MSFT > GOOGL > AAPL > AMZN(各7.5-9%) > TSM > TSLA > META(各4.5-6.5%) > AMD
- **选股逻辑**：AI 产业链全覆盖
  - 芯片设计：NVDA、AMD、AVGO
  - 云服务：MSFT(Azure)、GOOGL、AMZN(AWS)
  - 芯片代工：TSM
  - 未来关注：AI 应用层巨头（如 OpenAI 上市）

### 稳健型账户
- 蓝筹 + 成长混合配置
- 作为进取型和防守型之间的中轴

### 防守型账户（家庭/长期账户）
- **目标配置比例**：总资产的 **40%+**
- **持仓**：
  - 高股息个股：KO（可口可乐）、MO（奥驰亚）、JNJ（强生）、PG（宝洁）
  - 价值龙头：BRK（伯克希尔）
  - 红利 ETF：SCHD、XLP
  - 美债相关 ETF
- **核心逻辑**：稳定、保值、被动收入，不追求短期暴利

### 资金流动规则
```
进取型/稳健型收益达到高位 → 部分套现 → 利润转入防守型
                                     → 加固安全垫
                                     → 降低整体风险
```

## 四、买入策略：金字塔加仓法

### 规则定义
```
策略名称: PyramidBuy (金字塔加仓)
触发条件: 目标股票价格回调至预设区间
执行方式: 分批、由大到小逐步买入
目标效果: 拉低平均持仓成本
```

### 具体规则

#### R-BUY-01: 预设买入价格梯队
- 根据支撑位/估值，**提前设定** 3-5 个买入价格节点
- 价格从上到下，每层间距根据波动率调整
- 示例（Meta 实操）：
  - 第一档: $690（观察，不买）
  - 第二档: $650.5（触发买入）
  - 第三档: $627（加仓防守价）
  - 第四档: $596（深度防守价）

#### R-BUY-02: 仓位递减
- 第一笔买入金额最大，后续每档递减
- 逻辑：越跌越买，但每次买入量逐步减小
- 控制总投入不超过预算

#### R-BUY-03: 永不满仓原则
- **常规仓位上限: 70-75%**（保留 25-30% 现金）
- 极端行情（暴跌抄底）最多动用**现金储备的 50%**
- 始终留有余地应对更极端的情况

#### R-BUY-04: 危机加仓窗口
- 当市场因**短期外因**（非基本面）导致急跌时，识别为"买入窗口"
- 判断标准：基本面是否改变？
  - 未改变 → 加仓机会
  - 已改变 → 观望或减仓
- "19000 的纳指你追是价值投资；15000 的纳指（基本面没变）你不敢买，那不适合做投资"

## 五、卖出策略：倒金字塔减仓法

### 规则定义
```
策略名称: InversePyramidSell (倒金字塔减仓)
触发条件: 目标股票价格上涨至预设区间
执行方式: 分批、由小到大逐步卖出
目标效果: 逐步锁定利润，做零成本
```

### 具体规则

#### R-SELL-01: 预设减仓价格梯队
- 根据阻力位/目标价，**提前设定** 3-5 个减仓价格节点
- 每层减仓比例可相同或递增
- 示例（Meta 实操）：
  - $780 → 减仓 5%
  - $790 → 减仓 5%
  - $800 → 减仓 5%
  - $810 → 减仓 5%
  - 合计最多减 20%，目标"做零成本"
- 示例（NVDA 实操）：
  - $190 → 减仓 5%
  - $195 → 减仓 5%
  - $200 → 减仓 5%

#### R-SELL-02: 反弹减仓
- 暴跌期间买入的仓位，反弹后卖出 **20-30%** 以降低成本
- 不贪心等到最高点，有利润就部分锁定

#### R-SELL-03: 利润再分配
- 进取型/稳健型账户高位套现的利润
- 转入防守型账户，增加红利股/ETF 配置
- 目的：加固安全垫，降低整体组合风险

#### R-SELL-04: 情绪狂热时减仓
- 市场整体情绪过热 → 逐量减仓
- 降低成本、回笼资金
- 等待下一波机会

## 六、风险控制规则

### R-RISK-01: 仓位控制
```
常规总仓位: ≤ 75%
现金储备:   ≥ 25%
极端抄底时: 最多动用现金储备的 50%（即总现金降至 ~12.5%）
绝对底线:   始终保留可用现金
```

### R-RISK-02: 零杠杆原则
- **绝不使用融资/杠杆**
- 不借钱投资
- 管好现金流，不让自己轻易负债

### R-RISK-03: 攻守再平衡
```
触发条件: 进取型/稳健型账户收益显著（如 +30%+ 从成本）
操作: 高位套现部分利润 → 转入防守型
目标: 防守型配置 ≥ 总资产的 40%
频率: 跟随市场周期，非固定时间
```

### R-RISK-04: 安全边际控制
- 控制好仓位和资金比例
- 控制好成本（通过金字塔加仓拉低均价）
- 通过分批买卖控制变量，不一次性 all in 或 all out

### R-RISK-05: 区分暂时波动与基本面变化
```
判断框架:
├── 基本面改变（河流生态被破坏）→ 减仓/清仓
│   例: 美联储转向加息、公司核心业务崩塌、管理层重大丑闻
│
└── 短期扰动（鸭子搅浑了水）→ 持有/加仓
    例: 关税噪音、短期财报不及预期但业务健康、市场情绪恐慌
```

### R-RISK-06: 管理层风险监控
- 持续关注核心持仓公司的管理层变化
- CEO 变动、管理层丑闻 → 评估是否影响公司长期价值
- 案例：UNH CEO 事件导致股价下跌 10%+，但他判断长期影响有限，选择持有

## 七、投资品类偏好

### 两类核心个股
1. **改变未来的科技龙头** — NVDA、MSFT、GOOGL、AMZN、TSM 等
2. **稳定现金流的防御龙头** — KO、MO、JNJ、PG、BRK 等

### ETF 配置
| ETF | 类型 | 用途 |
|-----|------|------|
| SCHD | 高股息 | 防守型账户核心 |
| XLP | 必需消费品 | 防守型账户配置 |
| 宽基指数 ETF | 大盘跟踪 | 暴跌时快速建仓工具 |

### 明确不碰的
- 垃圾股、概念股
- 高杠杆产品
- 看不懂的公司
- A股（因市场生态原因）

## 八、交易执行纪律

### E-01: 提前挂单，不盯盘
- 所有买入/卖出价格**提前设定**
- 不靠盘中情绪做决策
- 触发则执行，不触发则等待

### E-02: 记录每一笔交易
- 公开记录买卖节点、仓位变化、操作逻辑
- 市场剧烈波动时在评论区实时更新操作
- 复盘是成长最快的方式

### E-03: 逆人性操作
- 别人恐慌时买入（有纪律的金字塔加仓）
- 别人贪婪时卖出（有纪律的倒金字塔减仓）
- 不跟风、不追高、不恐慌抛售

### E-04: 放眼长线
- 用长期视角去看问题
- 在长期时间轴上调整和优化投资操作
- 符合长期积累的特性

## 九、量化平台实现建议

以下是将上述原则转化为平台功能的映射：

| 原则/策略 | 平台功能模块 | 实现方式 |
|----------|------------|---------|
| 账户分层 | Portfolio Manager | 支持多账户/多策略组合，标记 attack/balanced/defense |
| 金字塔加仓 | Order Engine | 梯次限价单：自动计算价格梯队 + 递减仓位 |
| 倒金字塔减仓 | Order Engine | 梯次限价单：自动计算减仓价格 + 递增卖出量 |
| 仓位控制 | Risk Manager | 硬性仓位上限（75%），现金储备报警 |
| 零杠杆 | Risk Manager | 禁止 margin 交易，杠杆系数锁定 1x |
| 攻守再平衡 | Rebalancer | 当进取型收益超阈值 → 提示/自动转移利润到防守型 |
| 危机识别 | Opportunity Engine | 监控大盘跌幅，识别"短期扰动 vs 基本面变化" |
| 管理层监控 | Alert System | CEO 变动、重大管理层事件 → 告警 |
| 交易日志 | Trade Journal | 自动记录每笔交易 + 支持手动添加操作逻辑 |
| 情绪指标 | Market Sentiment | VIX/Fear&Greed → 狂热时提示减仓，恐慌时提示加仓 |
